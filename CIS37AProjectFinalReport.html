Abstract

For our project, we planned on creating a program that would act as a simple point-of-sale (POS) system c. We will list all possible items in a text file, and the program will read this text file. When a customer is ready for checkout, the program will allow the user to enter the barcode for each product, and it will calculate the total for all items in the cart. 

We used several concepts from Chapter 11, C File Processing. To read from our text file, we used functions like fgets and other basic file processing functions in the C standard libraries. We also utilized structs, arrays, and pointers in our C program in order to store our data and manipulate our program’s variables. In short, our program incorporates a variety of C programming features and constructs, including many conce pts introduced in class.

Not only did we strengthen our grasp of the course material through this project, but we also gained valuable experience through collaboration and exploration of new ideas. Although our project may be preliminary, it has given us a glimpse of how C programming would be used outside the classroom, leaving us with a greater appreciation of the C language and its relevance in today’s society.

Introduction

When we were first deciding on our topic, we wanted to choose something that would not be overly complicated, but still complex enough so that we would be able to have a satisfying challenge. After some deliberation, we decided to program a simple point-of-sale (POS) program, drawing inspiration from the cashiers we encounter during our day-to-day shopping purchases.

Obviously, because we do not have a barcode scanner, we knew that we could not make our program offer the exact same functionalities that commercial POS systems would implement. We decided to compromise by having the user manually enter the barcode of each item. Now, we needed to create a program that would store each item’s price and let a customer calculate the total of all items in his or her shopping cart.

Knowing that we could not possibly hardcode all the prices into our C source code, Van Tran suggested that we store all the prices in a text file, which the program will read to obtain the information it needs. At that point, our project focus boiled down to creating a program that would read this text file, gather the information it needs, and then help store customers proceed with checkout.

Our program explores:

  • How data files are processed by C programs
  • How to read data using sequential-access file processing.
  • How to apply C file processing to real-world systems (like markets, shopping centers, etc)

Implementation

The C program we created is designed to retrieve all item information from a text file, store that information in an array, and use the stored information to process customer purchases during checkout. We separated our program into two parts: processing the text file and calculating each customer's total transaction amount.

        We made the following assumptions in our project:

  1. The information inside our text file is well-formatted and listed in numerical order according to barcode. (However, the barcodes do not need to be consecutive.)
  2. No barcode can be zero or negative.
  3. No price can be zero or negative.
  4. For simplicity, the number of distinct items (i.e., the number of barcodes) is less than or equal to a symbolic constant, LIST_MAX, which is defined in the program code.

Overall Concept

  1. Create a file to store all the information of item (code, name, price).
  2. Use “r” mode to open and read the existing file.
  3. Use fgets to read data sequentially from the file.
  4. Create an array to store all the items in the file.
  5. Use fclose to close the file since we are not using it anymore.
  6. When the user enter the barcode in main, there are two cases:
  1. If the code entered does not match any of the codes in the file → ask user to try again.
  2. If the code match a code in the file → process the item
  1. Repeat step 6 until the customer has scanned each item.
  2. Repeat steps 6 and 7 for the next customer, and so on, until an employee quits the program.

The C Program

Description

        As mentioned, our program is divided into two portions: reading the text file and then handling customer transactions. At first, we were thinking of searching the text file each time the user enters a barcode; however, after John Lin recommended storing item information in an array, we decided to go with his idea. To do so, our program creates an Item array, opens the text file and reads its information into the array, and then closes the text file. While this process will result in an initial startup delay, especially if our text file lists many items, it is faster than our previous approach because the program does not need to search the text file each time a customer enters an item; it can just search the array.

        In our code, we defined a struct to represent an item. Our struct contains three fields: barcode, name, and price. We then defined type Item to refer to struct item.

After doing so, in the main function, we created an Item array allItems with size LIST_MAX, a symbolic constant we defined in the top of our code. (Once again, we made the assumption that there will be at most LIST_MAX distinct barcodes, so by creating an array of size LIST_MAX, we will have enough space to store all of our items. Naturally, if the store has less than LIST_MAX distinct items, then we will have unused space in our array.)

        Once it opens the file, our program proceeds to read each line. Here, we decided to have our program ignore lines that do not follow the format specified in the preceding section on our text file. That way, if our text file gets unintentionally corrupted, our program will not crash if it cannot properly parse the file. For each (properly formatted) line in the text file, the program will go to the next allItems entry and update its three fields. Once the program has obtained all the information in the text file, it closes the text file.

        When the program has all the information it needs, it will proceed with engaging customers in checkout. We designed the program to repeat the checkout process for each customer, looping infinitely until an employee enters the special exit passcode specified at the top of our code, defined in the symbolic constant QUIT_PASS. First, our program welcomes the user and asks for the first barcode. Assuming QUIT_PASS has not been entered, then program then searches allItems for the appropriate Item and asks the user how many he or she would like to buy. (For example, if the user wants to buy four boxes of cereal, he or she would enter the barcode for cereal, and then enter 5.) The program then displays the cost of buying the specified quantity and adds that cost to the total amount. This process repeats until the user has no more items left; at that point, the user should enter the letter x to quit. The program will then display the grand total for the customer's transaction, and it will repeat the entire process for the next customer.

        While handling customer transactions, we decided to make our program somewhat robust to handle improper customer input. In our code, if the customer does not enter a valid input, the program will have the user reenter his or her input. For example, if the customer enters an invalid barcode, we ask the customer to try again. We also check each scanf call to make sure the user's input is properly read; if not, then we have the user try again. Having such error-handling ensures that the program will not simply crash if the user enters improper input.

To search for items by barcode, we decided to have the program perform a binary search on allItems. Compared to searching each array element sequentially, a binary search is much faster because with each comparison, we can eliminate half of our possible values. Let our array Items be sorted by barcode. (This is why we made the assumption that the information in our text file is listed by barcode in numerical order. When we read each line's information into allItems, our array would necessarily be sorted by barcode.) We first find the middle value in our array, and we compare our barcode with the middle Item's barcode. If the barcode is smaller than our middle barcode, we repeat the binary search on the lower half of the array. Likewise, if the barcode is larger than our middle barcode, we repeat the binary search on the upper half of the array. If the middle barcode matches with the barcode we are looking for, then we return the index of that middle value so we can access that particular Item.

New and Important Functions

  • strtok - Split string into tokens. A sequence of calls to this function split str into tokens, which are sequences of contiguous characters separated by any of the characters that are part of delimiters.
  • atoi - Convert string to integer
  • strcpy - Copy string. Copies the C string pointed by source into the array pointed by destination, including the terminating null character.
  • atof - Convert string to double.
  • strcmp - Compare two strings. Compares the C string str1 to the C string str2.
  • fflush(stdin) - Delete standard input stream after using keyboard to enter character or string

                  to be negative, then we have an invalid item price.

/*

This function simply checks if its argument is equal to the special employee

password specified at the top of the code. If so, it will return 1, which means

that the two strings match. Otherwise, it will return 0.

*/

int checkForSentinel(char input[100]) {

           if (strcmp(QUIT_PASS, input) == 0) // strcmp returns 0 if strings match

                  return 1; // in this case, 1 signifies that the strings match

           return 0; // otherwise, return 0 -- note that no else statement is needed

}

/*

Given a specified barcode, the function searchByBarcode will use pointers to "return"

the index of the item with the specified barcode. This function will also return 0

for a successful search; otherwise, it will return -1 if the barcode cannot be found.

 

Note that searchByBarcode calls a recursive function to perform a binary search on

its array argument.

*/

 

// function to search an array

int searchByBarcode(int barcode, Item allItems[], size_t end, size_t* indexptr) {

           return recurse(barcode, allItems, 0, end, indexptr);

}

 

// the recursive function that actually contains the binary search algorithm

int recurse(int barcode, Item allItems[], size_t startIndex, size_t endIndex, size_t* indexptr) {

 

           if (startIndex > endIndex) {

                  return -1; // let -1 be an error code

           }

 

           size_t mid = (endIndex + startIndex) / 2; // find "midpoint"

 

           if (allItems[mid].barcode == barcode) {

                  *indexptr = mid;

                  return 0;

           }

 

           else if (allItems[mid].barcode < barcode)

                  return recurse(barcode, allItems, mid + 1, endIndex, indexptr); // find barcode in upper half

           else // allItems[mid].barcode is greater than barcode

                  return recurse(barcode, allItems, startIndex, mid - 1, indexptr); // find barcode in lower half

}

 

 

 

/*

This is a function that takes a specified item and asks how many the user wants to buy. It accepts

a pointer to the grand total variable, and once it calculates the amount spent on buying some number

of the specified item, it adds it to the grand total.

*/

void processItem(Item i, double* grandTotalPtr) {

           int quantity;

           double tot; // total from buying some quantity of Item i

           printf("\n=================\n\n");

           printf("Product: %s\nPrice: $%.2f\n", i.name, i.price);

           printf("How many do you want to buy? (Enter negative number to subtract from cart.) ");

 

           /*

           The function scanf returns an integer value indicating how many inputs were sucessfully read.

           Because we are only reading an integer, scanf should return 1. If it does not, then there is

           a problem. This while loop will have the user try again.

           */

           while (scanf("%d", &quantity) != 1) {

                  puts("Error reading input. (If this problem persists, call an employee.)");

                  printf("Note that you must enter an integer. Please try again: ");

                  fflush(stdin); // flush buffer to ensure user can enter new input

           };

           fflush(stdin); // flush buffer to ensure user can enter new input

 

           // calculate and add to grand total

           tot = i.price * quantity;

           *grandTotalPtr += tot;

 

           printf("Cost of %d items: $%.2f\n", quantity, tot);

           puts("\n=================\n");

}
